# AI Agent Coordination

This file declares the AI agent configuration for this repository. Both
**Copilot CLI** and **Codex** (VS Code Copilot) use a shared routing guide.

## Agents

### Copilot CLI

- **Invoked via**: Terminal command (`copilot ...`)
- **Best for**: Complex automation, git operations, CI/CD setup
- **Config**: `.github/copilot-instructions.md`

### Codex (VS Code Copilot)

- **Invoked via**: VS Code chat panel
- **Best for**: Live coding, refactoring, interactive SDD workflow
- **Config**: `.codex/skills/`

## Routing Matrix

| Category         | Task                        | Primary | Fallback | Notes                                                            |
| ---------------- | --------------------------- | ------- | -------- | ---------------------------------------------------------------- |
| **SDD**          | Create/update SDD specs     | Codex   | CLI      | Editor context preferred; CLI for batch doc updates              |
|                  | Implement feature from SDD  | Codex   | CLI      | Live coding preferred; CLI for multi-file refactor               |
| **Analysis**     | Repository analysis         | Codex   | CLI      | Workspace awareness preferred; CLI for complex grep              |
|                  | Search codebase             | CLI     | Codex    | Batch speed preferred; Codex for contextual search               |
| **Architecture** | Validate clean architecture | Codex   | CLI      | Interactive guidance preferred; CLI for systematic checks        |
|                  | Refactor parser             | Codex   | CLI      | Test-driven feedback preferred; CLI for large restructure        |
| **Webview**      | React/webview changes       | Codex   | CLI      | Component awareness preferred; CLI for multi-module edits        |
| **Automation**   | CI/CD setup                 | CLI     | Codex    | Script execution required; Codex if workflow guidance needed     |
|                  | Batch file edits            | CLI     | Codex    | Parallel ops preferred; Codex for interactive refinement         |
| **Complex Ops**  | Multi-slice refactor        | CLI     | Codex    | Git coordination required; Codex if per-step iteration preferred |

### How to Read This Matrix

- **Primary**: Recommended agent for this task (best capabilities/context match)
- **Fallback**: If Primary reaches token limit, network issues, or context loss, try this agent
- **Notes**: Why each assignment, when fallback might be better choice

### When to Use Fallback

✅ **Valid reasons to switch agents**:

- Token limit reached on Primary agent
- Primary agent session lost/timeout
- Task expanded beyond Primary's comfortable scope
- Need a different perspective (e.g., already tried one approach)
- Specific tool only Primary has (e.g., bash in CLI for automation)

❌ **Not a reason to switch**:

- Just prefer one agent personally (stick with Primary for consistency)
- Task is slightly outside Primary's wheelhouse (Primary can usually adapt)

## Source of Truth

**All agents reference these files** (never duplicate rules):

- `AGENTS.md` - **Authoritative** routing guide and architecture rules
- `docs/specs/` - SDD specifications and use cases
- `README.md` - Build/test commands

## How Agents Use This Repository

### Copilot CLI

1. Reads `.github/copilot-instructions.md`
2. Follows link to `AGENTS.md` § "AI Agent Routing Guide"
3. Looks up task type in routing matrix
4. Reads relevant `docs/specs/` for context
5. Executes with bash/git tools

### Codex

1. Reads `.codex/skills/*/SKILL.md`
2. Finds reference to `AGENTS.md` § "AI Agent Routing Guide"
3. Looks up task type in routing matrix
4. Reads relevant `docs/specs/` for context
5. Executes with editor integration

## Maintenance

- **Update routing?** Edit `AGENTS.md` § "AI Agent Routing Guide"
- **Update skills?** Modify `.codex/skills/*/SKILL.md` and add comment linking to `AGENTS.md`
- **Update CLI instructions?** Modify `.github/copilot-instructions.md`
- **Add new agent?** Add row to routing matrix in `AGENTS.md`, then link from here

**Important**: Keep `.agent.md` as a lightweight index. All detailed rules live
in `AGENTS.md`. This file's job is to declare that multi-agent coordination is
active and point to the source of truth.
