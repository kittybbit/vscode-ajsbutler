# AI Agent Coordination

This file declares the AI agent configuration for this repository. Both
**Copilot CLI** and **Codex** (VS Code Copilot) use a shared routing guide.

## Agents

### Copilot CLI
- **Invoked via**: Terminal command (`copilot ...`)
- **Best for**: Complex automation, git operations, CI/CD setup
- **Config**: `.github/copilot-instructions.md`

### Codex (VS Code Copilot)
- **Invoked via**: VS Code chat panel
- **Best for**: Live coding, refactoring, interactive SDD workflow
- **Config**: `.codex/skills/`

## Routing Matrix

| Category | Task | Agent | Why |
|----------|------|-------|-----|
| **SDD** | Create/update SDD specs | Codex | Editor context + multi-file coordination |
| | Implement feature from SDD | Codex | Live coding with immediate feedback |
| **Analysis** | Repository analysis | Codex (repo-analyse skill) | Workspace-aware |
| | Search codebase | CLI | Faster batch operations |
| **Architecture** | Validate clean architecture | Codex (clean-architecture-refactor skill) | Contextual understanding |
| | Refactor parser | Codex (parser-change skill) | Test-driven, interactive |
| **Webview** | React/webview changes | Codex (webview-change skill) | Component-aware |
| **Automation** | CI/CD setup | CLI | Script execution + git workflows |
| | Batch file edits | CLI | Parallel operations |
| **Complex Ops** | Multi-slice refactor | CLI | Multi-step coordination, git management |

## Source of Truth

**All agents reference these files** (never duplicate rules):

- `AGENTS.md` - **Authoritative** routing guide and architecture rules
- `docs/specs/` - SDD specifications and use cases
- `README.md` - Build/test commands

## How Agents Use This Repository

### Copilot CLI
1. Reads `.github/copilot-instructions.md`
2. Follows link to `AGENTS.md` § "AI Agent Routing Guide"
3. Looks up task type in routing matrix
4. Reads relevant `docs/specs/` for context
5. Executes with bash/git tools

### Codex
1. Reads `.codex/skills/*/SKILL.md`
2. Finds reference to `AGENTS.md` § "AI Agent Routing Guide"
3. Looks up task type in routing matrix
4. Reads relevant `docs/specs/` for context
5. Executes with editor integration

## Maintenance

- **Update routing?** Edit `AGENTS.md` § "AI Agent Routing Guide"
- **Update skills?** Modify `.codex/skills/*/SKILL.md` and add comment linking to `AGENTS.md`
- **Update CLI instructions?** Modify `.github/copilot-instructions.md`
- **Add new agent?** Add row to routing matrix in `AGENTS.md`, then link from here

**Important**: Keep `.agent.md` as a lightweight index. All detailed rules live
in `AGENTS.md`. This file's job is to declare that multi-agent coordination is
active and point to the source of truth.
