# plan-update

## Purpose

Write or refresh `PLANS.md` for the current task in a way that matches repository policy.

## Checklist

- describe the task in one sentence
- state why the work is needed
- list scope and non-goals explicitly
- name affected layers
- capture assumptions and key decisions
- define acceptance criteria and a concrete test plan
- note compatibility risks and rollback steps

## Rules

- keep the plan short and reviewable
- call out ambiguity explicitly
- do not claim tests were run before they were actually run
