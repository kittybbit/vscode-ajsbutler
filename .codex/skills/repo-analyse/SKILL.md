# repo-analyze

## Purpose

Understand the repository before making changes.

## Steps

1. Read `AGENTS.md`.
2. Read `package.json`.
3. Identify desktop/web entry points.
4. Identify parser-related modules.
5. Identify VS Code API boundaries.
6. Summarize risks before editing.

## Deliverable

Provide:

- relevant modules
- architectural risks
- test impact
- compatibility impact
